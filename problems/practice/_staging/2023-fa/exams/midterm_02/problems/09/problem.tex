\begin{prob}

    An ``isolated node'' in a graph is a node that has no edges attached to it.

    What is the worst case time complexity of an efficient algorithm that
    checks to see if a graph $G = (V, E)$ has an isolated node? You should
    assume that the graph is represented as an adjacency list.

    \begin{choices}
        \choice $\Theta(1)$
        \correctchoice $\Theta(V)$
        \choice $\Theta(E)$
        \choice $\Theta(V + E)$
        \choice $\Theta(V \cdot E)$
    \end{choices}


\end{prob}
